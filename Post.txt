I built Catslock: a small standalone Windows utility that turns CapsLock into a keyboard lock.

Double-tap CapsLock within 500 ms and Catslock turns on. While it is active, all keyboard input is silently swallowed by a low-level Windows keyboard hook. Double-tap CapsLock again to turn it off.

It is a brilliant systems programming exercise because the obvious version is not quite enough to work reliably out of the box. The details matter.

The implementation is intentionally small:

- C++ `WH_KEYBOARD_LL` global hook via `SetWindowsHookEx`
- Win32 message pump to keep the hook alive
- `requireAdministrator` UAC manifest for reliable global hook behavior
- CapsLock double-tap detection with `GetTickCount64`
- Physical CapsLock toggle correction when Catslock activates
- Simple state machine: `IDLE -> FIRST_TAP -> CATSLOCK_ON -> FIRST_TAP_OFF -> CATSLOCK_OFF`
- State written to `%TEMP%\catslock.state`
- PowerShell helper, `catslock.ps1`, for querying or forcing state from any terminal
- Named event, `CatslockToggle`, with a relaxed security descriptor so non-admin terminals can signal the elevated C++ process out-of-band

It was a useful little Windows systems exercise: hooks, message pumps, UAC, named-event security, state files, and keeping the behavior silent enough that it feels like a real lock rather than a modal app.
